# 深大羽协抽签工具

这是一个面向 Windows 电脑、源代码公开的羽毛球比赛抽签与对阵表生成工具，来源于深大羽协曾经使用的 `BFSZU.cpp` 控制台程序。2.0 版本的目标不只是“能随机分组”，而是让校园羽毛球赛事的抽签过程尽量接近正式竞赛规则，同时保留深圳大学校内赛事常见的跨校区预选、学院/学部回避、纸质打印流转等实际需要。

本项目不是 OSI 标准开源许可证项目，而是源代码公开、大学高校羽毛球社团非商业免费使用、商业用途需授权的双许可证软件。

## 2.0 重点

2.0 是一次规则化升级：

- 淘汰赛种子数量按《羽毛球竞赛规则2025（节选）》中“2.3.1 种子数”的分档限制处理。
- 淘汰赛种子签位采用规则汇总表中的 16、32、64、128、256 签位保护位置。
- 非 2 的乘方参赛数会按规则思想生成首轮赛和轮空位置，让每个小组都进入完整的 2 的乘方淘汰树。
- 循环赛使用矩阵式对阵表，保留胜场、净胜、名次列，并按轮转法生成赛程顺序。
- 同学院/同单位信息会参与分组回避；个人赛循环赛中，同单位对阵会优先排到靠前场次。
- GUI 只保留一种公开规则算法，移除旧 v1 算法选择，避免干事误选。
- 导出结果支持 Excel、JPG、极清透明 PNG、A4 PDF；图片和 PDF 均从 Excel 对阵表渲染，保证版式来源一致。
- A4 PDF 中淘汰赛支持手动行列分页，循环赛自动生成封面页，并让每个小组单独占一页。

详细规则对应关系见 [docs/rules-compliance.md](docs/rules-compliance.md)。

## 当前结构

- `src/BadmintonDraw.Core`：抽签核心算法，不依赖界面和 Excel。
- `src/BadmintonDraw.Excel`：参赛名单 Excel 导入、结果 Excel/图片/PDF 导出、名单模板生成。
- `src/BadmintonDraw.App`：Windows WPF 图形界面。
- `tests/BadmintonDraw.Tests`：基于 xUnit 的算法、导入、导出测试。
- `docs`：规则对应、算法、公平性、使用和构建说明。

## 快速使用

1. 打开 `BadmintonDraw.App.exe`。
2. 点击“生成名单模板”，填写参赛名单。
3. 回到软件，选择参赛名单 Excel。
4. 选择比赛模式、项目类型、小组数和随机数种子。
5. 如果是淘汰赛且小组数为 2 的 n 次幂，按需要选择“每组出线”或“决出冠军”；其他多组淘汰赛会默认按每组出线处理。
6. 点击“预览抽签”检查分组、首轮赛、轮空或直接晋级名单。
7. 在“导出为”下拉框选择 Excel、JPG、极清透明 PNG 或 A4 PDF 打印，再点击“选择路径并导出”。

Excel 导出的结果文件包含：

- `对阵表`：淘汰赛大表格对阵表，或循环赛矩阵与赛程顺序。
- `抽签设置与审计信息`：比赛设置、随机数种子、输入哈希、生成时间等。
- `原始名单`：按新版模板列名保留导入数据。

## 赛制说明

淘汰赛有两个目标：

- `每组出线`：每个小组独立打到 1 个出线名额，适合校区预选、分区预选、每组取 1 人/队的场景。
- `决出冠军`：用于最终要产生冠军的淘汰赛。

深圳大学常见场景是粤海校区和丽湖校区先分别预选，再集中到粤海进行后续比赛。例如粤海男单人数与丽湖男单人数约为 5:3 时，可以先把粤海分成 5 个小组、丽湖分成 3 个小组，各小组产生出线者。对于这种“小组数不是 2 的 n 次幂”的淘汰赛，程序默认每组出线，不强行生成数学上不自然的冠军签表。

当小组数是 2 的 n 次幂且大于 1 时，GUI 才显示“淘汰赛目标”选择。选择 `决出冠军` 时，程序会先让各小组产生出线者，再把这些出线者接入后续冠军淘汰段。

## 规则遵循与深大变通

程序遵循的核心规则包括：

- 种子数量分档：少于 16 人最多 2 个种子，16 至 31 人最多 4 个，32 至 63 人最多 8 个，64 至 128 人最多 16 个，超过 128 人最多 32 个。
- 种子位置保护：1 号种子进入签表顶部，2 号种子进入底部，后续种子进入对应的 1/4 区、1/8 区、1/16 区等保护区。
- 轮空分布：参赛数不是 2 的乘方时，第一轮设置轮空，并尽量在上下半区和各级分区中均衡。
- 同队/同单位回避：同单位参赛对象尽量均匀分布；循环赛中同单位比赛优先进行。
- 循环赛轮转：奇数参赛时补轮空位，使用轮转思想生成各轮比赛顺序。

面向深大校内赛事，程序做了几项明确变通：

- “学院/学部”被视作校内赛事中的同单位信息，用于替代正式赛事里的会员协会、队伍或报名单位概念。
- 程序不自动判定谁应成为种子。种子应由赛事组织方根据近期成绩、技术水平或校内规程先行确定，软件只校验和执行种子签位保护。
- 对于同一批次种子，程序采用可复现的稳定落位，而不是引入一个额外的隐式二次抽签。这样结果更容易被现场复核；如赛事规程要求批次内再抽签，可先通过公开方式确定种子序号。
- “首轮赛”用于描述为了凑成 2 的乘方正赛树而产生的第一轮比赛，避免把校内分组淘汰误写成正式国际赛事语境下的“预赛”。

## 公平性与审计

新版使用公开随机数种子。只要程序版本、参赛名单、比赛设置、淘汰赛目标和随机数种子完全一致，任何人在本地重新运行都可以得到同样的抽签结果。

结果文件会写入：

- 随机数种子
- 输入哈希
- 生成时间
- 参赛数量、种子数量、小组数量

建议现场在领队会或抽签环节公开确认名单、种子名单、比赛模式和随机数种子，再生成并公示结果 Excel。详见 [docs/fairness.md](docs/fairness.md)。

## 导出格式

JPG、PNG 和 PDF 会先生成 Excel，再从 Excel 的 `对阵表` 工作表渲染，因此 Excel 版式调整后其他格式会同步变化。

- Excel：完整可编辑工作簿，适合裁判长留档和后续填写结果。
- JPG：标准清晰度图片，适合快速分享。
- 极清透明 PNG：透明背景，程序会在小于 20MB 的前提下尽量提高分辨率。
- A4 PDF：适合打印。淘汰赛可按行列切片拼接；循环赛会生成封面页，后续每个小组单独一页。

## 开发环境

建议使用 Windows 电脑和 Visual Studio 2022：

- .NET 8 SDK
- Visual Studio 2022 的“.NET 桌面开发”工作负载

命令行构建：

```powershell
dotnet restore
dotnet build
dotnet test
dotnet run --project src/BadmintonDraw.App
```

macOS/Linux 可以查看和测试核心代码，但 WPF 图形界面只能在 Windows 上构建和运行。

详细构建和发布命令见 [docs/build.md](docs/build.md)。

## 历史来源

原始 C++ 控制台程序已经从当前工程中移除，避免干事误用旧入口。需要追溯历史实现时，可以查看原始仓库 [TonyHuang6666/Badminton-Draw](https://github.com/TonyHuang6666/Badminton-Draw) 中的 [`BFSZU.cpp`](https://github.com/TonyHuang6666/Badminton-Draw/blob/main/BFSZU.cpp)。

`BFSZU.cpp` 文件头部注释说明，原程序是 22 届深大羽协会长 Tony Huang 编写的“通用快速抽签分组程序”，目标是服务日常单项赛或团体赛抽签分组。注释同时说明，受 2023 年 4 月校长杯时间限制，旧版采用单文件控制台程序组织，未做 Windows 图形界面，也没有重点处理错误输入；其种子和轮空分布追求均匀随机，但尚未完全按《羽毛球竞赛规则》(2021) 中上下半区相关规定排布。当前 2.0 版本是在这个历史工具基础上，按《羽毛球竞赛规则2025（节选）》重新整理种子、轮空、循环赛和同单位回避逻辑，并补齐图形界面、Excel 导入导出、可复现随机数种子、名单校验、种子保护和审计信息。
