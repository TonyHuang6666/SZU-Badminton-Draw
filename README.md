# 深大羽协抽签工具

这是一个面向 Windows 电脑、源代码公开的羽毛球比赛抽签工具，来源于深大羽协曾经使用的 `BFSZU.cpp` 控制台程序。新版本的目标是让非技术干事也能完成比赛抽签，同时让全校同学可以公开查看抽签原理。

## 当前状态

仓库已经改造成分层的 Windows 图形化工具：

- `src/BadmintonDraw.Core`：抽签核心算法，不依赖界面和 Excel。
- `src/BadmintonDraw.Excel`：参赛名单 Excel 导入、结果 Excel/图片/PDF 导出、名单模板生成。
- `src/BadmintonDraw.App`：Windows WPF 图形界面。
- `tests/BadmintonDraw.Tests`：基于 xUnit 的基础算法和 Excel 导入导出测试。
- `docs`：算法、公平性和使用说明。

## 最终使用方式

1. 打开 `BadmintonDraw.App.exe`。
2. 点击“生成名单模板”，填写参赛名单。
3. 回到软件，选择参赛名单 Excel。
4. 选择比赛模式、项目类型、小组数和随机种子。
5. 如果是淘汰赛且小组数为 2 的 n 次幂，按需要选择“每组出线”或“决出冠军”；其他多组淘汰赛会默认按每组出线处理。
6. 点击“预览抽签”检查结果。
7. 在“导出为”下拉框选择 Excel、JPG、极清透明 PNG 或 A4 PDF 打印，再点击“选择路径并导出”。

Excel 导出的结果文件会包含：

- `对阵表`：淘汰赛对阵表或循环赛矩阵对阵表
- `抽签设置与审计信息`
- `原始名单`

JPG、PNG 和 PDF 会先生成 Excel，再从 Excel 的 `对阵表` 工作表渲染，因此 Excel 版式调整后其他格式会同步变化。PNG 使用透明背景，并在小于 20MB 的前提下尽量提高分辨率。

A4 PDF 的行为按比赛模式区分：

- 淘汰赛：支持手动设置行数和列数，用于打印拼接，并会按页面切片宽高自动选择竖向或横向 A4。
- 循环赛：自动按小组分页，每页尽量适配横向 A4 和极窄边距，不显示淘汰赛专用的行列分页输入。

## 淘汰赛目标

淘汰赛现在显式区分两个目标：

- `每组出线`：每个小组独立打到 1 个出线名额，适合校区预选、分区预选、每组取 1 人/队的场景。
- `决出冠军`：用于最终要产生冠军的淘汰赛。

当小组数不是 2 的 n 次幂时，程序默认并规范化为 `每组出线`，避免生成不成立的冠军签表。当小组数是 2 的 n 次幂且大于 1 时，GUI 才显示“淘汰赛目标”选择。选择 `决出冠军` 时，程序会先让各小组产生出线者，再把这些出线者接入后续冠军淘汰段。

例如 29 队分 8 组：

- `每组出线`：5 个 4 队组、3 个 3 队组，各组决出 1 个名额，共 8 个出线名额。
- `决出冠军`：先产生 8 个小组出线者，再进入 8 强淘汰赛，最终决出冠军。

## 公平性设计

新版使用公开随机种子。只要程序版本、参赛名单、比赛设置、淘汰赛目标和随机种子完全一致，任何人在本地重新运行都可以得到同样的抽签结果。

结果文件会写入：

- 随机种子
- 输入哈希
- 生成时间
- 参赛数量、种子数量、小组数量

## 开发环境

建议使用 Windows 电脑和 Visual Studio 2022：

- .NET 8 SDK
- Visual Studio 2022 的“.NET 桌面开发”工作负载

命令行构建：

```powershell
dotnet restore
dotnet build
dotnet test
dotnet run --project src/BadmintonDraw.App
```

macOS/Linux 可以查看和测试核心代码，但 WPF 图形界面只能在 Windows 上构建和运行。

详细构建和发布命令见 [docs/build.md](docs/build.md)。

## 历史来源

原始 C++ 控制台程序已经从当前工程中移除，避免干事误用旧入口。需要追溯历史实现时，可以查看原始仓库 [TonyHuang6666/Badminton-Draw](https://github.com/TonyHuang6666/Badminton-Draw) 中的 [`BFSZU.cpp`](https://github.com/TonyHuang6666/Badminton-Draw/blob/main/BFSZU.cpp)。

`BFSZU.cpp` 文件头部注释说明，原程序是 22 届深大羽协会长 Tony Huang 编写的“通用快速抽签分组程序”，目标是服务日常单项赛或团体赛抽签分组。注释同时说明，受 2023 年 4 月校长杯时间限制，旧版采用单文件控制台程序组织，未做 Windows 图形界面，也没有重点处理错误输入；其种子和轮空分布追求均匀随机，但尚未完全按《羽毛球竞赛规则》(2021) 中上下半区相关规定排布。当前 WPF 版本是在这个历史工具基础上，补齐图形界面、Excel 导入导出、可复现随机种子、名单校验、种子保护和审计信息。
