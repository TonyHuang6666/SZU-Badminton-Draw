# 深大羽协抽签工具

这是一个面向 Windows 和 macOS 的羽毛球赛事工作台，覆盖抽签、对阵表生成、赛程编排、现场记录表流转、多项目冲突检查和材料包导出，来源于深大羽协曾经使用的 `BFSZU.cpp` 控制台程序。

当前 4.2 版本的定位不再只是“生成抽签结果”，而是把校园羽毛球赛事从名单导入、身份核对、规则化抽签、对阵表导出，推进到多比赛日、多场馆、多场地、多项目协同编排、现场记录表流转、赛事存档和多格式交付。它既尽量恪守《羽毛球竞赛规则2025（节选）》中关于种子、轮空、循环赛和抽签变更的核心精神，也保留深圳大学校内赛事常见的跨校区预选、学院/学部回避、兼项冲突检查、纸质打印流转、裁判长现场审计等实际需要。

本项目不是 OSI 标准开源许可证项目，而是源代码公开、大学高校羽毛球社团非商业免费使用、商业用途需授权的双许可证软件。

## 4.2 重点

4.2 专注多项目赛程工作台的编排交互：

- Avalonia 多项目工作台取消“自动调整冲突”按钮；加载多个存档、切换 `紧凑完成` / `均衡宽松` / `决赛日友好` / `自定义` 策略，都会直接按当前策略全局重新编排。
- 默认三种策略以硬冲突 0 为生成目标；如果场地、时间、休息间隔或已完成锁定场次导致无解，程序会明确报错并回滚到上一可行赛程，不再把有硬冲突的方案当作成功结果。
- 自定义模式改为“当前操作锚点”模型：滑动某一天的负载率时，该日期成为主动目标，其他比赛日自动联动回填或释放；调整阶段波次或最小休息间隔时同样即时重排，失败即回滚。
- 自定义区域保留“恢复推荐分布”作为安全出口，可回到进入自定义时的系统推荐参数并立即重新编排。

## 4.1 重点

4.1 是一次面向裁判长现场工作台的打磨更新：

- 参赛名单模板新增“学号”和“搭档学号”，导入后用学号区分同名选手；普通赛程高级约束和多项目排程检查都会优先按学号判断同一名选手，同名但缺少唯一身份信息时会弹窗提醒，便于抽签前人工核对。
- “抽签”页新增参赛选手/队伍信息窗口，可浏览导入后的参赛库；窗口表格左侧带序号，支持横向/纵向滚动、缩小、还原和放大。
- 导入参赛名单后，可在参赛选手/队伍信息窗口直接修改“是否种子”和“种子序号”；两项都留空时按非种子处理，应用后需重新预览抽签。
- 单项目赛程窗口拖拽时改用结构化淘汰依赖校验，后序场次不能被拖到前序场次结束之前，避免旧版靠文字解析导致的漏判。
- 单项目“赛程预览”和多项目“时间 × 场地”总表已统一到底层 `ScheduleBoardView` 模型，日期、场地、时间槽、卡片状态、拖拽 payload、缩放边界和定位闪烁能力由同一套结构承载，减少 Avalonia/WPF 和单项目/多项目之间的行为漂移。
- 多项目全局编排器可锁定已完成场次，把所有未完成场次放入统一时间场地池，按淘汰依赖、场地占用、兼项选手时间和休息间隔重新排入可用位置。
- Avalonia 多项目工作台新增 `紧凑完成`、`均衡宽松`、`决赛日友好` 和 `自定义` 四种策略；加载多个存档后默认按“均衡宽松”直接化解冲突并生成合并赛程。
- 赛程预览新增高级约束提醒入口，按已确定风险、下一轮接续和推演风险组织提醒；已确定选手优先使用学号身份，点击提醒卡片可定位到赛程安排窗口中的对应场次。
- 校园赛/正式赛约束档会影响休息间隔、重点场次提醒和推演深度，既保留规则审计能力，也减少校园赛现场被理论风险淹没。

4.0 奠定了多项目赛事工作台基础：

- 发布 Avalonia 跨平台桌面版和 Windows WPF 版：macOS/Windows 可使用同一套核心工作流，Avalonia 版已加入深大羽协 logo 和正式跨平台界面口径。
- 支持选择多个 `.szbd` 赛事存档进入“多项目赛程编排”工作台：按场地和时间展示所有项目赛程，冲突卡片标红，裁判长可查看兼项明细、拖拽调整并保存回各项目存档。
- 多项目时间场地总表支持缩放、横向滚动和独立窗口查看；兼项明细支持按最短休息时间排序，调整后会实时重新检测严重冲突、间隔过短和同日提醒。
- 多项目严重冲突清零后可导出合并材料包，包含合并赛程记录表 Excel 和合并赛程安排表 Excel/PDF；另可导出多项目排程检查报告，作为赛前确认、现场交接和赛后复盘留档。
- 合并赛程记录表支持后续占空对阵自动公式更新，例如 `某场胜者/负者` 会随前序胜负自动刷新，方便裁判长现场填写和后续统计。
- 单项目“赛程预览”也支持打开时间场地窗口，未完成比赛可拖拽到空闲时间和场地；右侧列表和后续导出材料会同步更新。
- 首日/下一日/多项目合并材料包会先创建独立子文件夹，再导出记录表、赛程安排表、带时间场地对阵表、单场 PDF 计分表和团体赛记分表，避免现场文件散落。
- `.szbd` 赛事存档以 SQLite 事务保存完整抽签、赛程、累计结果、顺延补赛、导入日志和更正历史，并在更新前自动备份。
- 记录表导入会逐条列出所有待确认问题，并按记录表第一列“序号”定位；胜负冲突会阻止导入，比分、用时或比赛日更正由裁判长确认。
- PDF/图片导出使用内置中文字体，降低 Windows 端中文变方格的风险；单场比赛计分表每场一页横版，左侧预留 A4 板夹夹持空间。
- 继续保留规则化抽签、官方种子签位、轮空分布、循环赛矩阵表、多比赛日赛程、场馆场地预设和 Excel/JPG/PNG/PDF 多格式导出。

详细赛程设计见 [docs/scheduling.md](docs/scheduling.md)，规则对应关系见 [docs/rules-compliance.md](docs/rules-compliance.md)。

## 当前结构

- `src/BadmintonDraw.Core`：抽签和赛程编排核心算法，不依赖界面和 Excel。
- `src/BadmintonDraw.Excel`：参赛名单 Excel 导入、结果 Excel/图片/PDF 导出、名单模板生成、赛程表导出。
- `src/BadmintonDraw.Workflows`：跨 UI 复用的抽签、赛程工作流封装，供 WPF 和 Avalonia 逐步共用。
- `src/BadmintonDraw.App`：Windows WPF 图形界面。
- `src/BadmintonDraw.Desktop`：Avalonia 跨平台桌面版，用于同步开发 macOS/Windows/Linux 版本。
- `tests/BadmintonDraw.Tests`：基于 xUnit 的算法、导入、导出和赛程编排测试。
- `docs`：规则对应、算法、公平性、赛程编排、使用和构建说明。
- `samples`：可直接导入软件的虚拟测试名单。

## 快速使用

### 抽签与对阵表

1. 打开 `BadmintonDraw.App.exe`。
2. 点击“生成名单模板”，填写参赛名单；同名选手建议填写“学号/搭档学号”区分身份。
3. 回到软件，选择参赛名单 Excel。
4. 选择比赛模式、项目类型、小组数和随机数种子。项目类型通常会随名单自动识别为单打、双打或团体。
5. 如果是淘汰赛且小组数为 2 的 n 次幂，按需要选择“每组出线”或“决出冠军”；其他多组淘汰赛会默认按每组出线处理。
6. 点击“预览抽签”检查分组、首轮赛、轮空或直接晋级名单。
7. 在“导出为”下拉框选择 Excel、JPG、极清透明 PNG、A4 PDF 或“全部导出”，再点击“选择路径并导出”。

### 赛程编排

1. 先完成抽签预览。
2. 切换到“赛程编排”页。
3. 选择比赛日期、时间段、场馆和场地，点击“添加赛程日”。如果有多个比赛日，可重复添加。
4. 设置单场比赛预计占用时间、单名选手每日最多场次。
5. 点击“生成赛程预览”；如需查看或临时调整“时间 × 场地”表格，可在赛程预览上方点击“打开窗口”，拖动未完成比赛到空闲时间和场地，或右键卡片选择“移动到指定比赛日...”做跨日移动。
6. 点击“创建赛事存档”，生成 `.szbd` 单文件存档。以后可直接“打开赛事存档”恢复抽签、完整赛程、累计结果和导入日志，不需要重新选择历史记录表。
7. 点击“导出首日材料包”，一次生成首日记录表、赛程安排表、带时间场地对阵表和计分表。
8. 如需额外留档或分享完整赛程表，在“完整赛程表导出为”下拉框选择 Excel、JPG、极清透明 PNG、A4 PDF 或“全部导出”，再点击“导出完整赛程表”。
9. 当天比赛结束后，点击“导入记录表并导出下一日材料包”。程序会先事务更新赛事存档，再用累计结果刷新后续对阵并导出下一比赛日完整材料包。
10. 多个项目都创建赛事存档后，切换到“多项目赛程编排”菜单，选择编排策略并点击“加载多项目赛程”，选择至少两个 `.szbd`；程序会按当前策略直接全局编排未完成场次。右侧总表可查看冲突、缩放或打开独立窗口查看场地表，并用“兼项明细”核对跨项目选手；可手动拖拽微调，右键卡片跨日移动，也可切换策略或调整自定义参数重新生成。严重冲突为 0 后可导出合并材料包，确认后点击“保存调整到赛事存档”，也可导出当前排程检查报告留档。

### 跨平台桌面版

macOS/Windows/Linux 跨平台桌面版位于 `src/BadmintonDraw.Desktop`，使用 Avalonia，当前已支持名单模板、名单导入、抽签预览、多格式导出、多日赛程编排、首日/下一日材料包、带时间场地对阵表、赛事存档、记录表导入和多项目赛程编排工作台。仓库提供 `scripts/publish-macos.sh` 生成未签名 `.app` / `.dmg` 内部测试包；详细开发和运行说明见 [docs/macos.md](docs/macos.md)。

赛事文件职责：

- `.szbd`：程序可信存档，保存赛事状态、累计结果、导入日志和更正历史，不建议使用数据库工具手工修改。
- `.xlsx`：名单导入、现场填写、人工审计和跨人员交换。
- `.pdf` / `.png` / `.jpg`：打印、公示和分享。

存档更新会校验记录表中的隐藏赛事标识和场次标识。重复文件按 SHA-256 自动跳过；胜负方冲突会阻止导入；比分、用时或比赛日更正需要裁判长确认。确认框会完整列出所有问题，不只展示示例；记录表问题使用第一列“序号”定位。每次有效更新前会在存档同目录的 `Backups` 文件夹保留备份，最多保留最近 10 份。

## 导出结果

抽签结果 Excel 包含：

- `对阵表`：淘汰赛大表格对阵表，或循环赛矩阵与赛程顺序。
- `抽签设置与审计信息`：比赛设置、随机数种子、输入哈希、生成时间等。
- `原始名单`：按新版模板列名保留导入数据，包括学号和搭档学号，方便区分同名人员。

赛程表 Excel 包含：

- `赛程明细`：逐场列出日期、时间、场地、组别、阶段、双方和备注，适合裁判长审计。
- `时间场地网格`：按比赛日、时间段、场地铺开的现场执行表。
- `对阵记录表`：按比赛顺序列出当日场次，第一列为场次序号，预留比分、用时和可下拉选择的胜方；后续占空对阵可随前序胜负自动更新。导入记录表时，胜方是推进签表的依据；比分或用时缺失、比分方向与胜方不一致会弹窗逐条提醒，但可由裁判长确认继续。
- `赛程参数`：记录比赛日、时间段、场地、单场比赛预计占用时间和每日上限等参数。

首日和下一比赛日材料包会生成：

- `x月x日赛程记录表.xlsx`：现场填写入口，包含顺延补赛和隐藏赛事标识。
- `x月x日赛程安排表.xlsx` / `.pdf`：只包含对应比赛日的赛程明细、时间场地网格和参数。
- `x月x日带时间场地对阵表.xlsx` / `.pdf`：用于核对签表与赛程。
- `x月x日单场比赛计分表.pdf`：按新版羽毛球比赛记分表模板生成，每场比赛一页横版，左侧预留 A4 板夹夹持空间。表内“序号”与赛程记录表第一列一致，方便裁判长回收纸质计分表后快速定位、对照和统计赛果；未决出的占位场次保留姓名区空白。
- `x月x日团体赛记分表.xlsx`：仅团体项目生成，替代旧的 Word 团体记分表模板。

导出材料包时，软件会先在你选择的目录下创建 `x月x日材料包` 或 `x月x日多项目合并材料包` 子文件夹，再把所有文件放进去，避免多个 Excel/PDF 直接散落到桌面或下载目录。

多项目赛程编排工作台会读取多个 `.szbd` 存档，并在右侧生成按比赛日切换的“时间 × 场地”总表。第一行是场地编号，第一列是比赛时间；同一选手同时间冲突、休息间隔不足，以及同一场地时间重叠的比赛卡片会标红。左侧“兼项明细”会列出跨项目选手、涉及项目、未完成场次、冲突数量、最短休息间隔和逐场赛程，并可按最短休息时间排序；拖拽、策略切换或自定义参数调整后重新打开即为最新状态。右侧总表支持“打开窗口”，独立窗口内可切换比赛日和缩放，缩放可低于主界面的最小比例，场地较多时可放大窗口再横向滚动。

多项目加载后默认使用“均衡宽松”策略自动编排：程序会锁定已完成场次，把所有未完成比赛放入统一时间场地池，按项目内部淘汰依赖、场地占用、兼项选手时间冲突和最小休息间隔重新安排，并用每日目标负载率、阶段波次、跨项目统一收官日、少改原赛程、少跨日和重要场次靠后等软评分选择位置。收官日不再作为 GUI 可调项，而是内置在 `紧凑完成`、`均衡宽松` 和 `决赛日友好` 的默认评分中；裁判长如果需要个别调整，可在独立赛程窗口右键卡片跨日移动。左侧可以切换 `紧凑完成`、`均衡宽松`、`决赛日友好` 和 `自定义`，切换后会自动重新生成；自定义模式会显示每日负载率滑杆和阶段推进比例，每次调整都以当前控件为锚点联动其他参数并重新编排，失败会回滚到上一可行方案。严重冲突为 0 后可导出合并材料包，包含合并赛程记录表 Excel、合并赛程安排表 Excel/PDF。保存时会分别更新各项目 `.szbd`，并在对应目录创建备份。

导出的多项目排程检查报告会生成一个独立 Excel 文件。它不是 GUI 冲突处理入口，而是给裁判长、负责人和现场工作人员做赛前确认、交接和复盘留档，包含：

- `检查总览`：统计赛事数、总场次、已完成/未完成场次、比赛日、场地、兼项选手、冲突卡片、严重冲突、间隔过短、同日提醒和导出判断。
- `当前赛程卡片`：列出当前工作台里的每一场比赛、状态、冲突级别、时间、场地、双方和来源文件。
- `严重冲突`：同一选手在同一时间段跨项目参赛。
- `间隔过短`：两场跨项目比赛间隔低于界面设置的最小休息时间。
- `同日提醒`：同一选手同日参加多个项目但未低于最小休息间隔，用于裁判长人工确认体能和现场调度。
- `全部检查明细`：汇总所有检查项，方便筛选和复核。
- `输入赛事`：列出参与检查的赛事存档、项目类型、场次数和未决占位边数。

多项目编排只读取 `.szbd` 赛事存档。普通赛程高级约束、多项目工作台和排程检查报告共用同一套选手身份逻辑：非团体选手优先使用名单中的 `学号` / `搭档学号` 作为身份键，没有学号时才退回姓名，因此同名不同学号不会互相误报，同一学号会被识别为同一人。尚未由前序比赛决出的 `某场胜者/负者` 不会被误当成选手；若对应前序结果已经导入赛事存档，工作台和报告都会用累计结果解析后再检查。自动编排仍会使用结构化淘汰依赖处理这些未决占位，确保后续轮次不会早于前序比赛。已完成场次默认不允许拖动，避免和已导入赛果记录不一致。

抽签结果和完整赛程表支持四种导出格式；现场材料包固定采用可填写的 `.xlsx` 和适合打印的 `.pdf`，其中单场比赛计分表只生成 PDF，团体赛记分表只生成 Excel，不额外生成 JPG/PNG，避免现场文件过多。图片和 PDF 均先生成 Excel，再从 Excel 工作表渲染，因此 Excel 版式一旦调整，其他格式会同步变化：

- Excel：完整可编辑工作簿，适合留档、填写结果和后续人工调整。
- JPG：标准清晰度图片，适合快速分享。
- 极清透明 PNG：透明背景，程序会在小于 20MB 的前提下尽量提高分辨率。
- A4 PDF：适合打印。淘汰赛对阵表可手动行列切片；循环赛会生成封面并让每个小组单独一页；赛程表会按比赛日自动分页。
- 全部导出：一次生成 Excel、JPG、PNG、A4 PDF 四种格式。

## 赛制说明

淘汰赛有两个目标：

- `每组出线`：每个小组独立打到 1 个出线名额，适合校区预选、分区预选、每组取 1 人/队的场景。
- `决出冠军`：用于最终要产生冠军的淘汰赛。

深圳大学常见场景是粤海校区和丽湖校区先分别预选，再集中到粤海进行后续比赛。例如粤海男单人数与丽湖男单人数约为 5:3 时，可以先把粤海分成 5 个小组、丽湖分成 3 个小组，各小组产生出线者。对于这种“小组数不是 2 的 n 次幂”的淘汰赛，程序默认每组出线，不强行生成数学上不自然的冠军签表。

当小组数是 2 的 n 次幂且大于 1 时，GUI 才显示“淘汰赛目标”选择。选择 `决出冠军` 时，程序会先让各小组产生出线者，再把这些出线者接入后续冠军淘汰段。

## 规则遵循与深大变通

程序遵循的核心规则包括：

- 种子数量分档：少于 16 人最多 2 个种子，16 至 31 人最多 4 个，32 至 63 人最多 8 个，64 至 128 人最多 16 个，超过 128 人最多 32 个。
- 种子位置保护：1 号种子进入签表顶部，2 号种子进入底部，后续种子进入对应的 1/4 区、1/8 区、1/16 区等保护区。
- 轮空分布：参赛数不是 2 的乘方时，第一轮设置轮空，并尽量在上下半区和各级分区中均衡。
- 同队/同单位回避：同单位参赛对象尽量均匀分布；个人赛循环赛中，同单位比赛优先进行。
- 循环赛轮转：奇数参赛时补轮空位，使用轮转思想生成各轮比赛顺序。

面向深大校内赛事，程序做了几项明确变通：

- “学院/学部”被视作校内赛事中的同单位信息，用于替代正式赛事里的会员协会、队伍或报名单位概念。
- 程序不自动判定谁应成为种子。种子应由赛事组织方根据近期成绩、技术水平或校内规程先行确定，软件只校验和执行种子签位保护。
- 对于同一批次种子，程序采用可复现的稳定落位，而不是引入一个额外的隐式二次抽签。
- “首轮赛”用于描述为了凑成 2 的乘方正赛树而产生的第一轮比赛，避免把校内分组淘汰误写成正式国际赛事语境下的“预赛”。
- 赛程编排不替代裁判长或竞赛委员会的最终安排，而是在给定场地、时间和每日场次上限后，生成一份遵守对阵依赖、便于审计的初步赛程。

## 公平性与审计

新版使用公开随机数种子。只要程序版本、参赛名单、比赛设置、淘汰赛目标和随机数种子完全一致，任何人在本地重新运行都可以得到同样的抽签结果。

结果文件会写入：

- 随机数种子
- 输入哈希
- 生成时间
- 参赛数量、种子数量、小组数量

当前版本会把赛程参数、多项目调整结果和导入记录写入可审计文件，建议现场同时留存：

- 原始名单
- 抽签结果 Excel
- 赛程表 Excel
- 带比赛时间和场地的对阵表
- `.szbd` 赛事存档及其 `Backups` 备份
- 多项目合并材料包和多项目排程检查报告（如有）

建议现场在领队会或抽签环节公开确认名单、种子名单、比赛模式和随机数种子，再生成并公示结果 Excel。赛程表生成后，还应由裁判长人工审计关键轮次、每日场次上限、场地资源和比赛时间是否符合实际预约情况。详见 [docs/fairness.md](docs/fairness.md)。

- Visual Studio 2022 的“.NET 桌面开发”工作负载

## 历史来源

原始 C++ 控制台程序已经从当前工程中移除，避免干事误用旧入口。需要追溯历史实现时，可以查看原始仓库 [TonyHuang6666/Badminton-Draw](https://github.com/TonyHuang6666/Badminton-Draw) 中的 [`BFSZU.cpp`](https://github.com/TonyHuang6666/Badminton-Draw/blob/main/BFSZU.cpp)。

`BFSZU.cpp` 文件头部注释说明，原程序是 22 届深大羽协会长 Tony Huang 编写的“通用快速抽签分组程序”，目标是服务日常单项赛或团体赛抽签分组。注释同时说明，受 2023 年 4 月校长杯时间限制，旧版采用单文件控制台程序组织，未做 Windows 图形界面，也没有重点处理错误输入；其种子和轮空分布追求均匀随机，但尚未完全按《羽毛球竞赛规则》(2021) 中上下半区相关规定排布。当前版本是在这个历史工具基础上，按《羽毛球竞赛规则2025（节选）》重新整理种子、轮空、循环赛和同单位回避逻辑，并补齐图形界面、Excel 导入导出、多格式渲染、可复现随机数种子、名单校验、种子保护、单项目/多项目赛程编排、赛事存档和审计信息。
