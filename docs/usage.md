# 使用说明

本文面向实际办赛干事，说明如何从名单模板生成抽签结果、对阵表和赛程表。规则依据和程序设计边界见 [rules-compliance.md](rules-compliance.md)，赛程编排的专门说明见 [scheduling.md](scheduling.md)。

## 1. 准备名单

打开软件后点击“生成名单模板”，得到 `深大羽协参赛名单模板.xlsx`。

当前内置模板列为：

| 列名 | 用途 |
| --- | --- |
| 姓名 | 单打选手姓名；双打第一名选手姓名 |
| 学号 | 第一名选手学号，用于区分同名选手 |
| 学院/学部 | 单打、双打选手所属单位；团体赛时作为参赛队伍名称 |
| 搭档姓名 | 双打搭档姓名 |
| 搭档学号 | 双打搭档学号，用于区分同名选手 |
| 搭档学院/学部 | 双打搭档所属单位，导出原始名单时保留 |
| 是否种子 | 填“是”表示种子；不是种子可以留空或填“否” |
| 种子序号 | 可选，填写大于 0 的整数，例如 1、2、3、4 |
| 备注 | 可选，导出时保留 |

填写规则：

- 单打：填写“姓名”，建议填写“学号”，可填写“学院/学部”。
- 双打：填写“姓名”和“搭档姓名”，建议填写双方学号，可填写双方“学院/学部”。
- 团体赛：只填写 C 列“学院/学部”即可；模板备注中也写有“如为团体赛则仅填写C列学院/学部”。
- 旧版模板仍可导入，`搭档` 会被识别为 `搭档姓名`，`队伍` 会被识别为 `学院/学部`。

导入文件必须是 `.xlsx`。非 Excel 文件、损坏文件、加密文件或伪装成 `.xlsx` 的文件会被拒绝，并在界面底部显示错误。

## 2. 名单校验和提醒

导入名单时，程序会做以下硬性校验：

- 至少包含“姓名”或“学院/学部”列。
- 单打必须有“姓名”。
- 双打必须有“姓名”和“搭档姓名”。
- 团体赛必须能读到“学院/学部”；如果没有该列，才会退回使用“姓名”。
- `是否种子` 只能填“是”或“否”，也兼容 `true/false/1/0/yes/no`。
- `种子序号` 必须是大于 0 的整数，不能重复，不能大于当前参赛数量允许的种子数量。
- 种子数量按规则限制：16 人以下最多 2 个，16-31 人最多 4 个，32-63 人最多 8 个，64-128 人最多 16 个，129 人及以上最多 32 个。
- 如果填写了 `种子序号`，但 `是否种子` 填“否”，程序会要求修正。

以下情况只提醒，不阻止继续预览或导出：

- 同名选手。只要出现同名就会弹窗提醒，并列出姓名、搭档姓名中的位置以及已填写的学号。
- `是否种子` 填“是”但没有填写 `种子序号`。1 个时在左下角提醒，2 个及以上时弹窗列出所有未编号种子。

这些提醒的目的是提示名单可能需要补充或核对学号。程序会把学号保存到参赛选手/队伍信息中，普通赛程高级约束、多项目排程检查和导出材料都可以用它区分同名人员；如果你确认同名人员身份无误，可以继续点击“预览抽签”。

种子名单应由赛事组织方在抽签前确定。程序会执行种子数量和签位保护，但不会自动判断某名选手或队伍是否应该成为种子。

## 3. 自动识别项目类型

选择名单后，程序会根据内容自动切换项目类型：

- 发现“搭档姓名”或旧列“搭档”有内容：切换为双打。
- 有“姓名”且没有搭档：切换为单打。
- 没有姓名和搭档，只有“学院/学部”或旧列“队伍”：切换为团体。
- 如果团体名单同时填写队伍和联系人姓名，请先在界面选择团体赛模式，程序会保留团体赛识别。

## 4. 设置抽签

软件中需要设置：

- 比赛模式：单项赛淘汰赛、单项赛循环赛、团体赛淘汰赛、团体赛循环赛。
- 项目类型：单打、双打、团体；导入名单后通常会自动识别。
- 小组数：要分成多少个组。
- 随机数种子：可以现场生成，也可以由裁判长、参赛代表或公开方式共同确定。

淘汰赛还会根据小组数显示或隐藏“淘汰赛目标”：

- 小组数不是 2 的 n 次幂：不显示选择项，程序默认使用 `每组出线`。
- 小组数是 2 的 n 次幂且大于 1：显示 `每组出线` 和 `决出冠军` 两个选项。

`每组出线` 适合校区预选、分区预选、每组取 1 人/队的场景。`决出冠军` 适合最终要从所有参赛单位中产生冠军的场景；当选择多组决冠军时，程序会先让各组产生出线者，再进入后续冠军淘汰段。

当前 GUI 只有一种公开规则算法，不再提供旧版 v1 算法选择。

如果你正在处理粤海、丽湖等校区先分别预选的赛事，小组数可以直接理解为“出线名额数”。例如粤海分 5 组、丽湖分 3 组时，程序会让每个小组各自决出 1 个出线名额，而不是强行生成总冠军。

## 5. 预览抽签和导出对阵表

点击“预览抽签”后，右侧会显示：

- 总分组
- 第一轮名单
- 轮空或直接晋级名单

确认无误后，在左侧“导出为”下拉框中选择导出格式。默认选项是 Excel，最后点击“选择路径并导出”再选择保存位置。

- `Excel 文件 (*.xlsx)`：完整抽签结果工作簿。
- `JPG 图片 (*.jpg)`：从 Excel 对阵表渲染，清晰度保持标准导出。
- `极清 PNG（透明背景） (*.png)`：从 Excel 对阵表渲染为透明背景 PNG，程序会在不超过 20MB 的前提下尽量提高分辨率。
- `A4 PDF 打印 (*.pdf)`：从 Excel 对阵表渲染为 A4 页面。
- `全部导出`：使用你选择的文件名作为基名，一次生成 `.xlsx`、`.jpg`、`.png`、`.pdf`。

保存对话框会自动给出可读的默认文件名，格式大致为：

```text
赛事名_赛制_项目规模_小组数_淘汰赛目标_生成时间_seed随机数种子尾号.扩展名
```

例如 `深大羽协男单_淘汰赛_单打157人_8组_每组出线_20260605_1032_seed7801.xlsx`。循环赛没有“淘汰赛目标”字段。赛事名默认从导入名单文件名提取，程序会自动去掉“参赛名单”“模板”“159人”等弱信息，并清理 Windows 文件名不允许的字符。

A4 PDF 行为：

- 淘汰赛：预览抽签完成并选择 `A4 PDF 打印` 后，分页行列输入框才会出现。程序按“行/列”设置切片，适合打印后拼接；行列均为 1 时即整张表压缩到 1 页 A4。程序会按每页切片的宽高自动选择竖向或横向 A4，边距使用极窄设置。
- 循环赛：不会显示行列分页输入框。程序会生成 1 页封面总览，后续每个小组各自适配为窄边距横向 A4 页面。

循环赛导出会在矩阵中标出场次编号，并在每个小组矩阵下方生成“赛程顺序”。同学院/同单位之间的比赛会优先排到前面，便于现场先完成需要回避或提前处理的对阵。团体赛循环赛不显示“同单位优先”，因为每支队伍本身就是一个单位。

Excel 导出文件包含：

- `对阵表`
- `抽签设置与审计信息`
- `原始名单`

`原始名单` 子表使用新版列名：`姓名`、`学号`、`学院/学部`、`搭档姓名`、`搭档学号`、`搭档学院/学部`、`是否种子`、`种子序号`、`备注`。

## 6. 进入赛程编排

抽签预览完成后，可以切换到“赛程编排”页。赛程编排不是重新抽签，而是把已经生成的对阵表转为可执行的比赛安排。

赛程编排需要以下信息：

- 比赛日期。
- 当天可用时间段。
- 当天可用场馆和场地。
- 单场比赛预计占用时间。
- 单名选手每日最多场次。

软件内置了深圳大学常用场地：

- `运动广场东馆羽毛球场`：默认填入 B1-B8、C1-C8。
- `至快体育馆`：至快1 至 至快12。
- `至畅体育馆`：至畅1 至 至畅10。
- `全部`：合并以上场地。
- `自定义场地`：保留你手动输入的场地列表。

场地文本框可以手动删改，例如需要东馆 A、D 两排时可以直接补充；使用逗号、顿号、空格或换行分隔均可。

## 7. 添加多个赛程日

现实比赛往往不是一天打完。你可以为每一天单独设置日期、时间段和场地，然后点击“添加赛程日”。

例如：

| 日期 | 时间段 | 场馆 | 场地 |
| --- | --- | --- | --- |
| 2026-06-06 | 14:00-18:00 | 运动广场东馆羽毛球场 | B1-B8、C1-C8、D1-D8 |
| 2026-06-07 | 14:00-18:00 | 运动广场东馆羽毛球场 | B1-B8、C1-C8 |
| 2026-06-13 | 14:00-18:00 | 至快体育馆 | 至快1-至快6 |

如果同一天再次添加，程序会用新的设置替换原来的那一天。左侧表格中选中某一行后，可以点击“删除选中赛程日”。

## 8. 设置耗时和每日上限

`每场` 是单场比赛预计占用时间，应包含比赛、换场、登记、补水和必要休息缓冲。普通校园羽毛球赛常用 30 分钟；如果局数较少或项目节奏快，可以设为 20 或 25 分钟。

`单名选手每日最多场次` 用于控制运动员负荷：

- 1：非常宽松，适合高强度项目或更重视恢复。
- 2：常规推荐值。
- 3：场地或时间紧张时可用，但需要裁判长人工审慎确认。

如果资源不足以在这些限制下排完全部比赛，程序会提示资源不足，而不是强行生成不合理赛程。

淘汰赛选择“决出冠军”且已生成淘汰赛签表后，还可以启用赛程分界线。界面顺序是先选择“分界线”，再填写“分界线前设置”，最后填写“分界线后设置”。例如把分界线设为 `8强` 时，`16进8` 及更早阶段使用“分界线前设置”，`8进4`、半决赛和决赛使用“分界线后设置”。选择“不使用”时，所有比赛使用统一赛程设置。

## 9. 生成并导出赛程表

点击“生成赛程预览”后，右侧会显示逐场明细。程序会尽量在满足以下条件的情况下安排比赛：

- 后续轮次必须晚于前序比赛。
- 同一选手或队伍当天不超过设置的每日最多场次；非团体选手优先按学号识别，没有学号时才按姓名识别。
- 同一时间同一场地只能安排一场比赛。
- 阶段名称与对阵表表头保持一致。

右侧“赛程预览”上方可以点击“打开窗口”，在独立窗口中查看“时间 × 场地”表格。窗口内支持切换比赛日、缩小、恢复 100% 和放大；如果现场需要临时调整，可以把未完成的比赛卡片拖到当前比赛日的目标时间和场地，也可以右键卡片选择“移动到指定比赛日...”后直接指定目标日期、时间和场地。Avalonia 版拖动悬停时会实时校验目标格：绿色表示可直接放置，黄色表示可放置但会产生休息间隔等提醒，红色表示会造成场地重叠、淘汰依赖倒置等硬冲突并禁止放置。若移动的是后续轮次的前置场次，确认前会弹出“连锁移动预览”，列出受影响的直接和间接后续场次；裁判长可选择“只移动当前场次”，也可以选择“连锁移动后续场次”，让程序把同项目依赖链后移到最近合法空位。误拖后可在主界面或独立窗口点击“撤销调整”，回退上一步手动移动或整次连锁移动。调整后，逐场明细和后续导出的材料都会同步使用新的安排。

导出赛程时可以选择：

- `Excel`
- `JPG 图片`
- `极清 PNG（透明背景）`
- `A4 PDF 打印`
- `全部导出`

赛程 Excel 包含：

- `赛程明细`：完整逐场信息，保留“首轮赛1胜者”这类完整来源，便于审计。
- `时间场地网格`：按比赛日、时间、场地排布，适合现场执行。
- `对阵记录表`：按比赛顺序列出当日场次，第一列“序号”用于现场定位，预留比分、用时和可下拉选择的胜方；后续占空对阵可随前序胜负自动更新。记录表随首日/下一日材料包生成，默认文件名为 `x月x日赛程记录表.xlsx`。当天比赛结束后，可导入一张或多张记录表并导出下一比赛日材料包，程序会累计读取胜方并刷新后续对阵；比分、用时异常只提醒确认，未填胜方的场次可顺延到下一比赛日记录表和单场计分表 PDF。
- `赛程参数`：记录比赛日、场地、耗时、每日上限等输入。

赛程图片和 PDF 主要导出 `时间场地网格`。A4 PDF 会按比赛日自动分页，每个比赛日一页横向 A4，便于打印查看。

### 赛事存档

完整赛程生成后，建议立即点击“创建赛事存档”，保存为 `.szbd` 文件。该文件是程序继续赛事时的唯一可信状态，内部使用 SQLite 保存：

- 完整抽签结果、参赛单位和赛程快照。
- 已完成比赛结果。
- 顺延补赛列表。
- 已处理比赛日。
- 可由完整赛程和累计结果计算出的剩余待决场次。
- 导入文件名、SHA-256、导入时间、警告和更正次数。
- 比分、用时或比赛日更正前后的历史值。

后续打开软件时，可直接点击“打开赛事存档”。继续更新赛事时，不需要重新选择原始名单文件，也不需要每次重新选择全部历史记录表；但为了审计，仍建议把原始名单和导出的结果文件一起留档。

记录表仍使用 `.xlsx`，用于现场填写和交换。由赛事存档导出的新版记录表含隐藏赛事标识；误选其他赛事的记录表会被阻止。重复导入同一文件不会重复写入；胜负方冲突会强制拦截；同一胜负结果下的比分、用时或比赛日变化需要裁判长确认。确认窗口会逐条列出全部问题，并使用记录表第一列“序号”定位，而不是只给一条示例或显示 Excel 物理行号。

GUI 中显示的“待决”表示整份赛程中尚未产生结果的场次数，即 `赛程总场数 - 已完成结果数`。这和“顺延补赛”不是同一个概念：如果第一天所有比赛都打完，顺延补赛可以是 0，但后面比赛日尚未进行时，“待决”仍会大于 0。

每次有效更新前，程序会在 `.szbd` 同目录创建 `Backups` 文件夹，最多保留最近 10 份备份。

点击“导出首日材料包”或“导入记录表并导出下一日材料包”后，软件会让你选择一个文件夹，并一次生成对应比赛日需要的材料：

- `x月x日赛程记录表.xlsx`
- `x月x日赛程安排表.xlsx` 和 `.pdf`
- `x月x日带时间场地对阵表.xlsx` 和 `.pdf`
- `x月x日单场比赛计分表.pdf`，每场比赛一页横版；表内序号与记录表第一列一致，便于裁判长回收后快速定位、对照和统计赛果
- 团体项目额外生成 `x月x日团体赛记分表.xlsx`

软件会先在你选择的目录下创建 `x月x日材料包` 子文件夹，再把这些文件放进去；同名文件夹已存在时会自动追加序号。

## 10. 多项目赛程编排

当男单、男双、混双、团体等多个项目都已经创建 `.szbd` 赛事存档后，可以切换到“多项目赛程编排”菜单，把多个项目放进同一个赛程工作台统一检查和调整。

基本流程：

1. 分别为每个项目生成完整赛程，并创建 `.szbd` 赛事存档。
2. 如果某些后续轮次已经通过记录表导入产生真实胜者，先更新对应项目存档。
3. 在任一端软件中打开“多项目赛程编排”，设置最小休息间隔，默认 20 分钟。
4. 选择自动编排策略。默认是“均衡宽松”，加载后会直接尝试化解冲突并重新生成合并赛程。
5. 点击“加载多项目赛程”，选择至少两个 `.szbd` 存档。
6. 在右侧“多项目赛程”总表查看赛程。第一行是场地，第一列是时间；冲突或间隔过短的比赛卡片会标红。
7. 点击“兼项明细”查看跨项目选手清单。弹窗会列出每名兼项选手涉及的项目、场次数、未完成场次、严重冲突数、间隔过短数、最短休息间隔和逐场赛程，并支持按最短休息时间从短到长或从长到短排序；拖拽、切换策略或调整自定义参数后重新打开即为最新状态。
8. 场地较多时，点击右侧总表上方的“打开窗口”，在独立大窗口中切换比赛日、缩放并横向滚动查看所有场地。
9. 如需人工调整，先在右侧选择目标比赛日，再把未完成比赛卡片拖到目标时间和场地格子；若要跨日移动，可在独立窗口右键卡片选择“移动到指定比赛日...”，直接指定目标日期、时间和场地。Avalonia 版拖动悬停时会用绿色、黄色、红色提示目标格是否可放，红色目标会阻止落下并在状态栏说明原因。移动前序场次时会按该项目内部淘汰树弹出连锁影响预览，并单列已确定兼项选手在其他项目中的同日重叠、休息不足或体能提醒；未决占位不会展开所有理论晋级可能。裁判长可只移动当前卡片，也可选择连锁移动同项目后续依赖；连锁移动不会自动移动其他项目的无关场次，但会用全局场地、兼项和休息约束避开它们。程序会立刻重新检测，冲突解除后红色标记会消失；误拖后点击“撤销调整”即可回退上一步手动移动。
10. 如需重新交给程序处理，直接切换编排策略，或在“自定义”模式中调整每日负载率和阶段波次。程序会从加载时的基准赛程重新出发，锁定已完成场次，把所有未完成场次放入统一时间场地池，按项目内部淘汰依赖、场地占用、兼项选手时间冲突和最小休息间隔全局重新编排。
11. 严重冲突调整为 0 后，可以点击“导出合并材料包”，生成合并后的赛程记录表 Excel，以及合并赛程安排表 Excel/PDF，方便裁判长统一打印和填写。
12. 确认后点击“保存调整到赛事存档”。程序会分别更新对应 `.szbd`，并在原文件同目录创建备份。
13. 需要赛前确认、现场交接或赛后复盘留档时，点击“导出排程检查报告”。如果当前已经加载工作台，导出的是当前调整后的状态；如果尚未加载，程序会先根据所选 `.szbd` 构建一次多项目工作台再导出。

工作台和排程检查报告按三类列出：

- 严重冲突：同一选手在同一时间段有两场跨项目比赛。
- 间隔过短：两场比赛不重叠，但间隔低于最小休息时间。
- 同日提醒：同一选手同日多项目参赛，间隔达到要求，但仍建议裁判长确认体能和现场调度。

同一场地在同一时间段被多场比赛占用时，工作台也会把相关卡片标红。已完成场次不允许拖动，避免和已导入赛果不一致。工作台摘要会同时显示兼项选手人数、冲突数量和当前缩放比例，便于裁判长快速判断当前视图状态。

自动编排策略分为四种：`紧凑完成` 会尽量沿用原时间并尽快排完；`均衡宽松` 是默认策略，会在场地充足时平衡每日负载、阶段推进和收官日观感，避免第一天爆满而最后一天空闲；`决赛日友好` 会更积极地把半决赛、决赛和名次赛留到最后一天；`自定义` 会展开均衡宽松背后的参数，包括每个比赛日的目标场地负载率和阶段波次推进比例。跨项目统一收官日不再作为 GUI 可调选项，而是作为默认策略评分项保留；裁判长需要微调某些收官场次时，使用独立窗口的右键跨日移动。自定义调整遵循“当前操作锚点”模型：例如滑动某一天的负载率时，该日期成为主动目标，其他日期被动释放或回填；调整阶段波次也会即时重排。若当前时间、场地或休息间隔无法容纳全部未完成场次，程序会回滚到上一可行赛程并在状态栏提示，需要裁判长增加比赛日、场地、时间段，或放宽休息间隔后再调整。切换策略、调整自定义参数、修改最小休息间隔和保存到赛事存档都会清空手动移动撤销记录；“撤销调整”只用于回退最近的手动拖拽或右键移动。

跨项目选手冲突只检查已经确定具体选手/队伍的场次。非团体选手优先使用名单中的 `学号` / `搭档学号` 作为身份键，没有学号时才退回姓名；因此同名不同学号不会互相误报，同一学号跨项目会被识别为同一人。`某场胜者`、`某场负者` 等未决占位不会被误报。若前序比赛结果已经导入赛事存档，后续场次会按累计结果解析后参与检查。自动编排仍会使用这些场次的结构化淘汰依赖，保证未决占位对应的后续轮次不会早于前序比赛结束。

团体项目当前按队伍/参赛单位名称检查，不读取队伍内部名单。因此团体赛与个人项目之间的同名队伍/选手冲突需要裁判长结合报名表人工确认。

## 11. 带比赛时间和场地的对阵表

带比赛时间和场地的对阵表随材料包固定导出，不再需要单独勾选。软件会在对阵表对应格子里写入比赛时间和场地，例如：

```text
胜者1
2026-06-06 14:30-15:00
C1
```

这个文件适合裁判长拿来审计：

- 对阵树是否与赛程一致。
- 后续轮次是否安排在前序比赛之后。
- 哪一场比赛对应哪个时间和场地。
- 现场纸质计分表回收后，应把结果填到哪个格子。

材料包中的带时间场地对阵表固定导出 Excel 和 A4 PDF。完整赛程表单文件导出仍支持 Excel、JPG、PNG、A4 PDF 四种格式，主要用于留档、公示或临时分享。

## 12. 复现结果

若需要复查抽签结果，保留以下信息：

- 原始名单 Excel
- 比赛模式
- 项目类型
- 小组数
- 淘汰赛目标
- 随机数种子
- 输出文件中的输入哈希

若需要复查赛程表，额外保留：

- 比赛日列表
- 每天的时间段
- 每天的场地列表
- 单场比赛预计占用时间
- 单名选手每日最多场次
- `.szbd` 赛事存档和 `Backups` 中的最近备份
- 每天回收并导入过的 `x月x日赛程记录表.xlsx`

当前抽签规则固定在程序版本中。任何人使用同一版本程序、同一份名单和同一组设置重新生成，都应得到相同抽签结果。赛程编排还依赖场地和时间参数，因此复查赛程时必须同时确认这些参数。

如果抽签或排程后发现名单错误、退赛、场馆调整或赛事规程变更，建议保留原结果文件，并由裁判长决定是重新抽签、重新排程还是人工调整。软件不会自动替代裁判长处理赛后或赛中变更。
