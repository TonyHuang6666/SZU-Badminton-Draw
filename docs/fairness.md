# 公平性与公开审计

## 为什么需要随机种子

传统程序只显示“随机结果”，但外部同学无法复现随机过程。新版使用公开随机种子，让抽签过程可以被重复验证。

## 复现条件

以下内容完全一致时，结果应完全一致：

- 参赛名单
- 种子设置
- 比赛模式
- 项目类型
- 小组数
- 淘汰赛目标
- 随机种子
- 程序版本

淘汰赛目标会影响签表结构，因此也是复现条件的一部分。比如 29 队分 8 组时，`每组出线` 和 `决出冠军` 会得到不同的对阵表终点。

## 输入哈希

软件会根据名单和设置计算输入哈希，并写入结果 Excel。输入哈希用于证明复查时使用的是同一份输入。

输入哈希覆盖比赛模式、项目类型、小组数、公开算法版本、淘汰赛目标和名单内容。只要这些内容改变，哈希也会改变。

## 建议的现场流程

1. 公示参赛名单和种子名单。
2. 现场确认比赛模式、小组数和淘汰赛目标。
3. 现场确定随机种子，例如由裁判长和参赛代表共同给出一串字符串。
4. 软件生成抽签结果。
5. 公布结果 Excel 中的随机种子、输入哈希和生成时间。
6. 任何同学都可以下载源代码和名单复现结果。

## 种子保护

种子序号不是普通备注。程序会按 `SeedRank` 保护高位种子，使其尽量落在不同的小组位置；当种子数量超过小组数时，同组内的多个种子也会继续在对阵树中隔开。在淘汰赛切分附加赛时，程序会优先让种子轮空或直接进入正赛。这个规则更接近正式羽毛球赛事中“按种子序号放置签位并优先保护轮空”的思路。

## 赛制解释

程序支持校内常见的预选赛语义：多个小组可以只用于产生出线名额，而不直接决出总冠军。为了避免误解：

- 非 2 的 n 次幂小组数默认每组出线。
- 2 的 n 次幂小组数可以选择每组出线，也可以选择小组出线后继续决出冠军。

这种显式选择比单纯依靠小组数推断更透明，便于赛前公示和赛后复核。
