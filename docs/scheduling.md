# 赛程编排说明

3.0 新增的赛程编排功能，用于在抽签和对阵表生成之后，把比赛安排到具体日期、时间段和场地。它面向的是校园羽毛球赛事的现场执行：裁判长需要知道每一场何时打、在哪片场打、胜者进入哪里，干事需要打印时间场地表和计分表，参赛者需要看到一个逻辑上合理的赛程。

## 解决的问题

抽签结果只回答“谁和谁打、胜者去哪里”。赛程表还要回答：

- 哪一天打。
- 哪个时间段打。
- 哪片场地打。
- 后续比赛什么时候才可以开始。
- 同一名选手一天最多打几场。
- 裁判长如何快速根据纸质计分表更新后续对阵。

3.0 的赛程编排就是把这些信息组织成可执行表格。

## 基本流程

1. 完成名单导入和抽签预览。
2. 切换到“赛程编排”页。
3. 添加一个或多个赛程日。
4. 设置赛程分界线、分界线前后单场占用时间、每日最多场次。
5. 生成赛程预览。
6. 导出赛程表。
7. 如需审计和后续更新，同步导出带比赛时间和场地的对阵表。

赛程编排不会重新抽签，也不会改变对阵树。它只是把已经存在的对阵关系安排到时间和场地上。

## 多比赛日

现实比赛经常不是一天打完。软件允许每个比赛日独立设置：

- 日期
- 开始时间
- 结束时间
- 场馆
- 场地

这适合以下情况：

- 第一天和第二天场地多，后期比赛少，场地变少。
- 粤海、丽湖或不同场馆开放时间不同。
- 需要把半决赛、决赛安排到另一天。
- 场馆申请结果发生变化，需要重新生成赛程。

如果同一天重复添加，新的设置会覆盖旧设置。这样可以快速修正某一天的场地或时间。

## 场馆和场地

软件内置深圳大学常用场地：

| 场馆 | 场地 |
| --- | --- |
| 运动广场东馆羽毛球场 | 默认填入 B1-B8、C1-C8 |
| 至快体育馆 | 至快1-至快12 |
| 至畅体育馆 | 至畅1-至畅10 |
| 全部 | 合并以上所有场地 |

也可以选择“自定义场地”，手动输入任意场地名。场地之间可以用逗号、顿号、空格、分号或换行分隔。

场地文本框可以手动删改，例如需要使用东馆 A、D 两排时，可以直接补充 `A1-A8` 或 `D1-D8`。场地名称会写入赛程表和带比赛时间场地的对阵表，因此建议使用现场干事熟悉的短名称，例如 `C1`、`D8`、`至快3`。

## 分段耗时参数

大规模淘汰赛常见做法是前期比赛采用较短赛制，后期比赛采用更完整的赛制。例如 100 多人的项目，可以把 `8强` 设为赛程分界线。这里的“每场”是场地被占用的完整时间，应包含比赛、换场、登记、补水和必要休息缓冲：

- `8强前`：256进128、128进64、64进32、32进16、16进8 等阶段，可设置为每场 20 分钟、单名选手每日最多 3 场。
- `8强后`：8进4、4进2、决赛等阶段，可设置为每场 30 分钟、单名选手每日最多 2 场。

如果选择“不分段（统一设置）”，所有场次仍使用同一套耗时和每日最多场次。

使用分段耗时时，软件会按每片场地的下一次可用时间动态排程；20 分钟场次和 30 分钟场次可以自然混排，不再强制使用固定长度时间槽。

## 单名选手每日最多场次

这个参数用于防止程序为了排完比赛而让同一名选手一天打太多场。

- `1`：非常宽松，适合高强度项目或对恢复要求高的比赛。
- `2`：推荐默认值，适合多数校园羽毛球比赛。
- `3`：时间或场地紧张时可以使用，但建议裁判长人工复核。

对双打而言，一对选手作为一个参赛对象；对团体赛而言，一支队伍作为一个参赛对象。当前软件按参赛对象控制每日场次。

启用赛程分界线时，分界线前和分界线后分别使用各自的每日最多场次。名次附加赛不纳入分界线表述；排程上强制不晚于冠军决赛。

## 淘汰赛依赖逻辑

淘汰赛有天然依赖关系。比如：

```text
A组64进32第3场
A组128进64第5场胜者 vs A组128进64第6场胜者
```

这场 `64进32` 必须等两个 `128进64` 来源比赛安排完后才能安排。软件会维护可安排比赛队列，只有前置依赖已经排入赛程的比赛，才会进入下一批。

这避免了明显错误：

- 同一名选手同一时间出现在两场比赛。
- 首轮赛还没打完，后续轮次已经开始。
- 决赛和早期轮次被安排在同一个时间段。

如果可用比赛日、时间段、场地或每日最多场次不足，软件会提示资源不足，而不是生成逻辑错误的赛程。

资源不足时，软件仍会保留赛程预览：已经安排的场次照常显示，未安排的场次会以特殊颜色标记，并列出阶段、场次、双方来源和无法安排原因。未安排场次存在时，不支持导出赛程表或带时间场地的对阵表；需要先增加比赛日、场地、时间段，或调整每日最多场次后重新生成完整赛程。

名次附加赛会被视为颁奖前必须完成的名次决定场次。只要仍有 3/4 名赛、5-8 名半决赛、5/6 名赛或 7/8 名赛未排入赛程，冠军决赛不会被排到更早的时段；如果场地足够，冠军决赛可以和最后一批名次附加赛同一时段完成。

## 阶段名称

赛程阶段以前面对阵表表头为准：

- `首轮赛`
- `128进64`
- `64进32`
- `32进16`
- `16进8`
- `8进4`
- `4进2`
- `决赛`
- `出线`
- `3/4名赛`
- `5-8名半决赛`
- `5/6名赛`
- `7/8名赛`

这样做的好处是，对阵表、赛程明细和时间场地网格使用同一套语言。裁判长审计时，不需要再判断“这个 16进8 是小组内的，还是整项赛事的”。

## 赛程明细

`赛程明细` 工作表按逐场比赛列出：

- 序号
- 比赛日
- 时间
- 场地
- 组别
- 阶段
- 场次
- 选手/队伍A
- 选手/队伍B
- 备注

这里保留完整来源，适合审计。例如：

```text
D组64进32第3场
D组128进64第5场胜者
D组128进64第6场胜者
```

如果需要追查某场比赛为什么被安排到某个时间，优先看 `赛程明细`。

## 时间场地网格

`时间场地网格` 工作表更适合现场执行。它按比赛日分段，横轴是场地，纵轴是时间。

为方便裁判长快速填写，网格里会使用短引用：

```text
D组64进32第3场
15:00-15:30 C6胜 vs 15:00-15:30 C7胜
```

这种写法比完整的“D组128进64第5场胜者”更容易在同一张表里找到来源比赛。跨比赛日时会加日期，例如：

```text
6/7 15:00-15:30 C6胜
```

不同阶段会使用不同底色。早期轮次、正赛阶段、半决赛、决赛等颜色不同，方便肉眼扫表。

名次附加赛中可能出现“某场负者”。时间场地网格会用 `负` 作为短引用，例如 `15:00-15:30 C6负`，与正赛晋级的 `胜` 区分。

## 带比赛时间和场地的对阵表

赛程导出时可以同步导出带比赛时间和场地的对阵表。它会在原对阵表的比赛格中加入：

```text
胜者1
2026-06-06 14:30-15:00
C1
```

这个文件的用途是：

- 把抽签对阵树和赛程表连起来。
- 让裁判长知道每个晋级格对应哪一场具体比赛。
- 方便根据纸质计分表填写结果。
- 为未来“导入结果并生成下一比赛日赛程”打基础。

如果导出赛程时选择“全部导出”，这个带时间场地的对阵表也会同步导出 Excel、JPG、PNG、A4 PDF。

## A4 PDF

赛程表的 A4 PDF 与淘汰赛对阵表不同。赛程表不要求用户输入几行几列，而是按比赛日自动分页：

- 每个比赛日一页横向 A4。
- 使用窄边距。
- 主要呈现 `时间场地网格`。

这样更符合打印和现场使用方式：一天一页，干事拿着当天的场地时间表即可执行。

## 人工审计建议

软件生成赛程后，建议至少检查：

- 决赛、半决赛是否过早。
- 后续轮次是否确实晚于前序比赛。
- 同一参赛对象一天场次是否符合预期。
- 重点场次是否需要安排到更适合观赛或执裁的场地。
- 场馆开放时间和实际申请结果是否一致。
- 是否需要给跨项目参赛者额外休息时间。

如果发现需要调整，可以先修改赛程参数重新生成；如果只是少量场次需要人工移动，应保留调整说明。

## 当前边界

- 当前赛程功能不自动处理跨项目冲突。例如同一名同学同时参加男双和混双，需要组织者人工协调。
- 当前不自动生成单场计分表；这是后续功能方向。
- 当前不自动导入比赛结果并更新下一比赛日赛程；带时间场地的对阵表已经为这个方向预留了数据形态。
- 当前不自动考虑裁判数量、观众动线、直播场地等软约束。
