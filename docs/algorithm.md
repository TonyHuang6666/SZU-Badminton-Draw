# 抽签算法说明

## 背景

原始 `BFSZU.cpp` 使用控制台输入名单，通过随机编号、均匀分组、种子分散和首轮轮空名单计算，帮助深大羽协快速完成比赛抽签。

新版保留历史算法的思路，同时把规则拆成可审计的核心算法。旧 C++ 文件不再保留在当前工程中，需要追溯时可查看 Git 历史或 GitHub 原仓库。

## 数据模型

每个参赛单位会被转成一个 `DrawParticipant`：

- 单打：一个选手。
- 双打：一对选手，显示为 `[选手A 选手B]`。
- 团体：一个队伍。
- 种子：通过 `IsSeed` 和 `SeedRank` 标记。

算法内部处理的是参赛单位列表，不直接依赖 Excel 或界面。

## 通用分组步骤

1. 将非种子参赛单位按随机种子稳定洗牌。
2. 将非种子均匀切成指定小组数，各组人数差不超过 1。
3. 将种子参赛单位按随机顺序轮流放入各组，避免种子集中。
4. 对每组内部再次洗牌。
5. 输出总分组。

同一份名单、同一组设置、同一个随机种子，会得到相同的分组结果。

## 循环赛

循环赛只需要总分组结果。组内循环赛程可由人工或后续模块继续生成。

## 淘汰赛：公开规则算法

程序固定使用 `PerGroupPowerOfTwo`。它对每个小组独立计算第一轮人数：

```text
groupSize = 小组人数
base = 不超过 groupSize 的最大 2 的幂
第一轮人数 = 2 * (groupSize - base)
轮空或直接晋级人数 = groupSize - 第一轮人数
```

例子：

| 小组人数 | base | 第一轮人数 | 轮空或直接晋级人数 |
| --- | ---: | ---: | ---: |
| 4 | 4 | 0 | 4 |
| 5 | 4 | 2 | 3 |
| 6 | 4 | 4 | 2 |
| 7 | 4 | 6 | 1 |
| 8 | 8 | 0 | 8 |

这个算法的好处是每个小组都能解释为独立淘汰树。

旧版 `BFSZU.cpp` 的历史算法不再作为 GUI 选项保留。新的比赛统一使用公开规则算法，便于解释、复核和跨校区预选赛使用。
