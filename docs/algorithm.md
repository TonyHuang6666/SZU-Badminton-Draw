# 抽签算法说明

## 背景

原始 `BFSZU.cpp` 使用控制台输入名单，通过随机编号、均匀分组、种子分散和首轮轮空名单计算，帮助深大羽协快速完成比赛抽签。原始仓库见 [TonyHuang6666/Badminton-Draw](https://github.com/TonyHuang6666/Badminton-Draw)。

新版保留历史算法的思路，同时把规则拆成可审计的核心算法。旧 C++ 文件不再保留在当前工程中，需要追溯时可查看 Git 历史或 GitHub 原仓库。

## 数据模型

每个参赛单位会被转成一个 `DrawParticipant`：

- 单打：一个选手。
- 双打：一对选手，显示为 `[选手A 选手B]`。
- 团体：一个队伍，通常来自名单模板的“学院/学部”列。
- 种子：通过 `IsSeed` 和 `SeedRank` 标记。
- 选手所属单位和搭档所属单位：作为名单元数据保留，导出原始名单时写回。

算法内部处理的是参赛单位列表，不直接依赖 Excel 或界面。

## 通用分组步骤

1. 将非种子参赛单位按随机种子稳定洗牌。
2. 将非种子均匀切成指定小组数，各组人数差不超过 1。
3. 将种子参赛单位按 `SeedRank` 从高到低放入受保护的小组位置，例如 4 个小组时为第 1、4、2、3 组，尽量避免高位种子过早相遇。
4. 对每组内部再次洗牌。
5. 输出总分组。

同一份名单、同一组设置、同一个随机种子，会得到相同的分组结果。

## 循环赛

循环赛会导出矩阵式 `对阵表`。个人赛循环赛和团体赛循环赛本质相同，只是参赛单位名称不同：个人赛显示选手或组合名，团体赛显示队伍或学院名。

循环赛表格中，对角线为空轮，右侧保留胜场、净胜、名次列，方便现场填写或后续统计。

## 淘汰赛：公开规则算法

程序固定使用 `PerGroupPowerOfTwo`。它对每个小组独立计算第一轮人数：

```text
groupSize = 小组人数
base = 不超过 groupSize 的最大 2 的幂
第一轮人数 = 2 * (groupSize - base)
轮空或直接晋级人数 = groupSize - 第一轮人数
```

例子：

| 小组人数 | base | 第一轮人数 | 轮空或直接晋级人数 |
| --- | ---: | ---: | ---: |
| 3 | 2 | 2 | 1 |
| 4 | 4 | 0 | 4 |
| 5 | 4 | 2 | 3 |
| 6 | 4 | 4 | 2 |
| 7 | 4 | 6 | 1 |
| 8 | 8 | 0 | 8 |

这个算法的好处是每个小组都能解释为独立淘汰树。

切分第一轮时，程序会优先把种子放入轮空或直接进入正赛的位置。只有当某个小组的种子数量超过可用轮空名额时，多出来的种子才会进入第一轮附加赛。

如果一个小组内有多个种子，程序会继续在该组对阵树中按保护槽位放置种子。例如 8 个正赛槽位时，1、2、3、4 号种子会优先落在第 1、8、4、5 个槽位，尽量把高位种子隔开。

## 淘汰赛目标

淘汰赛通过 `KnockoutGoal` 明确目标：

- `OneQualifierPerGroup`：每组出线。每个小组独立打到 1 个出线名额，最终不再跨组决冠军。
- `Champion`：决出冠军。用于最终需要产生冠军的赛制。

当小组数不是 2 的 n 次幂时，程序会把 `Champion` 规范化为 `OneQualifierPerGroup`。原因是非 2 的 n 次幂小组数无法自然接入后续完整淘汰段，强行生成会出现不合理的 `7进3`、`3进1` 等轮次。

当小组数是 2 的 n 次幂且大于 1 时，程序允许两种目标：

- 选择 `每组出线`：例如 29 队分 8 组，最终得到 8 个小组出线名额。
- 选择 `决出冠军`：先让 8 个小组各出线 1 队，再接入 8 强、4 强、决赛，最终产生冠军。

## 对阵表连接线

导出的淘汰赛 `对阵表` 使用空白单元格边框绘制连接线。3 人组、5 人组这类有附加赛的小组会先把附加赛胜者接入正赛位置，再与轮空或直接晋级者连接到下一轮。

当不同小组人数不一致时，表头会按最大组内签表统一列数。例如 29 队分 8 组时，既有 3 队组也有 4 队组，表头会显示 `4进2`、`2进1`、`出线`。3 队组虽然只有 2 个正赛签位，也会直接画完整连接到 `出线` 列，避免出现缺线或孤立横线。

## 历史算法

旧版 `BFSZU.cpp` 的历史算法不再作为 GUI 选项保留。新的比赛统一使用公开规则算法，便于解释、复核和跨校区预选赛使用。
