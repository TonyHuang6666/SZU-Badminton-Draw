# 羽毛球竞赛规则对应关系

本文说明当前版本如何依据《羽毛球竞赛规则2025（节选）》组织抽签、赛程编排和多项目冲突审计，以及哪些地方是面向深圳大学校内赛事做出的明确变通。

## 规则依据

当前实现主要参考以下规则点：

- 循环赛：规则 1.2 附近给出了单循环赛的轮转法说明，并指出奇数参赛时可补“0”作为轮空；同组内同单位运动员之间的比赛应首先进行。
- 分组循环赛：规则 1.4 说明参赛人（对、队）数较多时可采用分组循环赛，组数确定后可抽签分组，也可用蛇形排列。
- 淘汰赛轮空：规则 2.2 说明当参赛数不是 2 的乘方数时，第一轮应有轮空，轮空数为下一个较大的 2 的乘方数减去参赛数，并应在上下半区和各级分区中均衡分布。
- 淘汰赛附加赛：规则 2.1 将淘汰赛场数表述为“人（对、队）数－1+附加赛场数”；规则 2.4 说明单淘汰赛只能产生第一、二名，如需排出后续名次，应另行增加附加赛。
- 种子数量：规则 2.3.1 规定种子数量分档。
- 种子位置：规则 2.3.2 规定 1 号种子在顶部、2 号种子在底部，3/4、5-8、9-16 等进入对应保护分区，并指向种子和轮空位置汇总表。
- 同队运动员：规则 2.3.3 要求同队运动员按均匀分布原则进入 1/2、1/4、1/8 等不同分区；同一会员协会运动员第一轮尽可能不相遇。
- 预赛抽签：规则 2.5.1 说明预赛也应按相关方法确定种子和抽签，预赛出线运动员进入正赛规定位置。
- 团体赛场序：团体赛规程允许采用三场制或五场制，并明确裁判长可根据运动员兼项情况调整场序；同一运动员同一轮团体赛不应在同一项目重复出场。
- 竞赛日程：竞赛日程应在充分利用场地的同时兼顾运动员合理负担；正式规则对单项运动员每日、每单元参赛场数和比赛间隔提出原则性限制。
- 抽签变更：规则“五、抽签变更”强调抽签结束后原则上不得随意更改，除非出现错误、退赛、严重不平衡等特殊情况。

赛程编排通常需要由竞赛组织方结合场地、裁判、项目数量、运动员负荷和赛事日程安排。软件的赛程功能不声称替代正式竞赛规程，而是在已确定的抽签结果和场地资源约束下，生成可审计、可调整、可留档的执行方案。

## 程序直接遵循的部分

### 种子数量分档

程序按参赛数量限制种子数量：

| 参赛数量 | 程序允许的最多种子数 |
| ---: | ---: |
| 少于 16 | 2 |
| 16 至 31 | 4 |
| 32 至 63 | 8 |
| 64 至 128 | 16 |
| 超过 128 | 32 |

如果参赛数量本身小于规则上限，程序不会允许种子数超过实际参赛数量。

### 种子签位保护

程序内置 16、32、64、128、256 签位的种子位置表。典型位置包括：

| 签位规模 | 高位种子保护位置 |
| ---: | --- |
| 16 | 1、16、5、12 |
| 32 | 1、32、9、24、5、13、20、28 |
| 64 | 1、64、17、48、9、25、40、56、5、13、21、29、36、44、52、60 |
| 128 | 1、128、33、96、17、49、80、112、9、25、41、57、72、88、104、120 |
| 256 | 1、256、65、192、33、97、160、224、17、49、81、113、144、176、208、240 等 |

这对应规则中“顶部/底部、1/4 区、1/8 区、1/16 区”的保护思想。对于小于 16 的小组，程序使用同样的上下半区保护原则，让 1 号和 2 号种子尽量远离。

### 轮空和首轮赛

规则要求非 2 的乘方参赛数在第一轮设置轮空。程序对每个淘汰赛小组独立计算：

```text
base = 不超过小组人数的最大 2 的乘方
首轮赛人数 = 2 * (小组人数 - base)
轮空或直接进入正赛人数 = 小组人数 - 首轮赛人数
```

例如 5 人组会安排 2 人打首轮赛，3 人轮空或直接进入正赛；7 人组会安排 6 人打首轮赛，1 人轮空或直接进入正赛。这个结果与规则中 5、6、7、8 人淘汰赛的轮空思想一致。

### 名次附加赛

单淘汰赛主树只自然产生冠军和亚军。若抽签前选择 `只排 3/4 名` 或 `排 3-8 名`，程序会在主对阵表下方增加“名次附加赛”区块，并在赛程中追加对应场次。3/4 名赛由两场半决赛负者参加；3-8 名附加赛还会安排 5-8 名半决赛、5/6 名赛和 7/8 名赛。

这部分不改变正赛抽签和种子保护位置，而是按规则 2.4 的含义作为“另行增加的附加赛”处理。若冠军主树不足 8 个签位，`排 3-8 名` 会退化为可执行的 3/4 名赛；不足 4 个签位时不生成名次附加赛。

### 循环赛矩阵和轮转法

程序为循环赛生成矩阵对阵表，右侧保留胜场、净胜、名次列。每个小组下方会生成赛程顺序，奇数参赛时按轮转法自然形成轮空。个人赛中如果同组内有同学院/同单位参赛对象，相关对阵会优先排到靠前场次，呼应规则中“同单位比赛先进行”的要求。

### 抽签公开和复核

规则强调抽签结果不应随意更改。程序通过随机数种子、输入哈希、生成时间和原始名单子表，帮助现场把“抽签前确认”和“抽签后复核”固定下来。只要名单、设置、程序版本和随机数种子一致，抽签结果可以复现。

## 赛程编排与规则边界

竞赛规则明确规定了抽签、种子、轮空、循环赛轮转、团体赛场序、竞赛日程和抽签变更等关键原则，但具体到校园赛事中的赛程表，仍需要结合现场资源安排。当前赛程编排遵循以下边界：

- 不改变已经生成的抽签结果。
- 不跳过对阵依赖，后续轮次必须晚于前序比赛。
- 不在同一时间同一场地安排多场比赛。
- 通过“单名选手每日最多场次”和跨项目最小休息间隔提醒控制运动员负荷，并支持按淘汰赛分界线设置前后阶段不同耗时；单项目 4 场/天是深大校园赛早期短局制下的变通上限，高规格赛事应优先控制在 3 场/天以内。
- 名次附加赛作为颁奖前需要决出的名次场次，排程上不晚于冠军决赛。
- 将赛程参数写入导出文件，便于事后复核。
- 多项目工作台会把严重冲突和警告标红，并可在锁定已完成场次的前提下全局重排未完成场次；已确定跨项目选手同日最多 6 场作为遵从规则负荷精神的硬约束，未决淘汰路径则按胜负分支生成负荷推演提醒，严重冲突未消除时禁止导出合并材料包。
- 生成的是可执行初稿，仍应由裁判长或竞赛组织方审定。

这意味着软件可以帮助组织者避免明显的逻辑错误，例如首轮未完成就安排决赛；但不能替代组织者判断是否需要为重点比赛留出休息时间、是否要错开同学院志愿者值班、是否要配合场馆临时开放安排。

正式规则中的休息间隔要求适用于高规格赛事；深大校内赛事常受场馆申请时段、课程安排、裁判数量和打印流转限制影响。软件因此采用“硬冲突强制、休息不足提醒、裁判长最终确认”的设计：同一选手时间重叠、同场地时间重叠、裁判容量并发超限、不可用场地时段被占用和淘汰依赖顺序错误属于必须解决的问题；间隔不足、同日多项、重点比赛错峰等会进入提醒或自动调整约束，最终仍由裁判长结合现场资源确认。

## 深大校内赛事的变通

### 学院/学部作为同单位

正式规则中的“同队”“同一会员协会”在校园赛事中通常对应学院、学部、校区或报名单位。程序把名单中的“学院/学部”和“搭档学院/学部”作为同单位信息，用于分组回避和循环赛同单位先赛。

这不是把学院等同于正式国际赛事的会员协会，而是把规则中的回避精神移植到深圳大学校内竞赛语境。

### 校区预选默认每组出线

深圳大学常见流程是粤海校区、丽湖校区分别预选，再集中进行后续比赛。例如粤海与丽湖人数比例为 5:3 时，往往希望粤海分 5 组、丽湖分 3 组，各组产生 1 个出线名额。

因此当淘汰赛小组数不是 2 的 n 次幂时，程序默认 `每组出线`，不强行生成 `7进3`、`3进1` 这类不符合完整淘汰树语义的冠军段。当小组数是 2 的 n 次幂时，程序才允许选择 `每组出线` 或 `决出冠军`。

### 首轮赛不是正式“预赛”

规则中“预赛”涉及进入正赛位置、候补、晋级和抽签发布等正式赛事流程。深大日常比赛中，很多小组内只是为了把 5 人、6 人、7 人调整到 4 人或 8 人正赛树而需要先打一轮。

程序把这类比赛称为“首轮赛”，避免误导为正式意义上的预赛。Excel 中的黄色格表示首轮赛胜者填入正赛位置，绿色格表示直接进入正赛。

### 同批次种子的稳定落位

规则中 3/4 号、5-8 号等同批次种子通常还要在对应保护区内抽签。程序为了公开复现，采用稳定的种子序号落位：种子序号一旦确定，落位也随之确定。

这让校内现场更容易复核，也避免“抽签程序内部又做了一次看不见的批次内抽签”。如果某项赛事规程要求同批次种子必须再次抽签，建议先在领队会公开完成同批次排序或抽签，再把结果填入“种子序号”。

### 团体赛同单位先赛不适用

团体赛中每个参赛对象本身就是一个单位，因此“同单位对阵优先”没有实际意义。程序只在个人赛循环赛中显示并使用同单位先赛提示；团体循环赛只生成普通轮转赛程。

### 深圳大学场地预设

软件内置运动广场东馆羽毛球场、至快体育馆、至畅体育馆等场地预设，是为了贴合深圳大学校内办赛流程。这些预设不是规则的一部分，也不影响抽签公平性；其他学校或社团可以使用自定义场地完成同样的赛程编排。

## 程序没有自动完成的部分

- 不自动判定谁是种子。种子应由赛事组织方依据校内规程、近期成绩或技术水平确定。
- 不自动处理退赛后的递补和重抽。抽签后如需变更，应由裁判长按赛事规程决定。
- 不自动计算循环赛最终名次。表格保留胜场、净胜、名次列，方便裁判长或后续统计模块填写。
- 不自动保证所有现实约束都最优，例如具体裁判轮换、裁判资质/回避、重点场次观赏性、选手不可用时间、场地临时不可用时间、正式赛事休息间隔和团体赛内部出场名单。
- 不自动决定团体赛三场制或五场制的具体上场名单；如后续接入团体赛阵容表，也应保留裁判长按兼项情况调整场序的入口。
- 不替代正式竞赛规程。软件是抽签、制表和赛程初排工具，最终解释权仍应属于赛事组织方和裁判长。
